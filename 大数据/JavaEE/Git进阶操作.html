git add . 与 git add -u的区别:
git add . ：他会监控工作区的状态树，使用它会把工作时的 所有变化提交 到暂存区，包括文件内容修改 (modified)以及新文件 (new)，但不包括被删除的文件。
git add -u ：他仅监控 已经被add的文件 （即tracked file），他会将被修改的文件提交到暂存区。)

git rm的用法:
git rm file_path 删除暂存区和分支上的文件，同时工作区里的文件也会删除
git rm --cached file_path 只删除暂存区或分支上的文件, 工作区里依然保留该文件, 只是不希望被版本控制(如果是删除整个文件夹用git rm -r --cached directory)

git reset用法:
git reset --soft HEAD~1
撤销最近一次的commit(撤销commit，不撤销git add)
git reset --mixed HEAD~1
撤销最近一次的commit(撤销commit，撤销git add)
git reset --hard HEAD~1
撤销最近一次的commit(撤销commit，撤销git add，工作区的代码改动将丢失, 上面两个工作区代码的更改将会保存。操作完成后回到上一次commit状态)
HEAD~1的意思是最近一个版本，也可以写成HEAD^。如果需要撤回最近两次提交的commit，可以使用HEAD~2，依次类推。如果希望回退到指定版本, 可以通过git log 找到提交记录, 选择该提交记录的id, 将HEAD~1替换为该id,如下


git status 先看一下add 中的文件
git reset HEAD 如果后面什么都不跟的话 就是上一次add 里面的全部撤销了
git reset HEAD XXX.py 就是对某个py文件进行撤销了
git reset HEAD file 即使对file文件夹进行撤销


git rm 与 git reset的区别:
git rm：用于从工作区和索引中删除文件
git reset：用于将当前HEAD复位到指定状态。一般用于撤消之前的一些操作(如：git add,git commit等)。

git push -f : 强制提交。当因为版本回退等各种原因导致本地仓库版本比远程仓库低时, 可使用。

显示本地的所有分支(只显示本地有的, 远程有的不会都显示出来):
git branch

切换分支(只有该分支存在或在本地存在才能切换过去):
git checkout 分支名

将master分支上最新的代码更新到其他分支上, 比如说某个个人分支:
1. 将分支切换到master
git checkout master

2. 将代码pull到本地
git pull

3. 修改冲突
4. 提交代码到本地仓库(会提交到master, 如果没有冲突则可以不提交, 不然的话提交会没有内容)
git add -u
git commit -m "Merge brach 'master' into yibo"


5. 切换到要更新的分支上
git checkout 分支名

6. merge
git merge master

7. 将本地跟新好的内容push到分支
git push
